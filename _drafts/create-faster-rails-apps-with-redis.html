How long are you willing to wait for a website to load? I know that's contingent on what you're waiting on but imagine it's Amazon. According to an article published last year on BBC half of us are only willing to wait 3 seconds for a shopping website to load. What does that mean for websites we build that aren't established and are trying to gain traction? I'd venture to say most people won't even wait 3 seconds so that begs the question of how do we make faster websites? The simple answer is caching but I'm going to show you how to use the in-memory data store Redis as a solution.

What is Redis? The first place you can find info is their website redis.io, but for our purposes, it's a caching store. If you go to the website and feel overwhelmed that's exactly what happened to me. The possibilities are vast but we'll keep it simple for this post.

What you'll need to Begin

  • Ruby 2.4.0 +
  • Rails 5.1.0 +
  • homebrew
  • Lenix (I'll be using macOS)

Installing Redis

Using homebrew it's extremely easy to add Redis. Just type the following into the terminal.

brew install redis
redis-server

That should start up the Redis server in your terminal and you'll see this hopefully.
